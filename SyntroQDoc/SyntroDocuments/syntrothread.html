Detailed Description

SyntroThread is alightweight wrapper for QThread.

yntroThread is a thin wrapper layer around Qts QThread. It provides a Windows-style message passing queue and associated functions. The normal sequence of events is to create a SyntroThread, perform any external initialization that might be required and then to call resumeThread(). To kill the thread, call exitThread().

SyntroThread also processes QThread timer events and maps them to SyntroThread messages with a message code of SYNTROTHREAD_TIMER_MESSAGE. These can be handled in the processMessage() function. The timer ID is passed in the intParam of the message to allow multiple timers to be supported.

SyntroThread would normally be subclassed to provide customized thread functionality.

Member Function Documentation

SyntroThread::SyntroThread(QString threadName = "Thread")

This is the constructor for SyntroThread. It may be called with threadName, a name for the thread  this can be useful for debugging.

void SyntroThread::exitThread() [virtual]

exitThread() should be called when the component wishes to shutdown the thread prior to exit. This function should be overriden to perform any clean up that the thread requires. The default exitThread() provided by SyntroThread includes these two lines:

  • m_run = false;
  • exit();

If this function is overridden, these two lines should be added to the overriding function.

void SyntroThread::initThread() [virtual protected]

intiThread() is called as part of the resumeThread() processing and should be used for any initialization that the thread needs to perform prior to normal execution.

bool SyntroThread::isRunning()

isRunning() will return true if the thread is still running, false if it has exited the run loop.

void SyntroThread::postThreadMessage(int message, int intParam, void * ptrParam) [virtual]

This function can be called to place a message for thread on its queue. A message consists of three values:

  • message. This is a code that identifies the message. See SyntroThread.h for examples.
  • intParam. This is an integer parameter that is passed with the message.
  • ptrParam. This is a pointer parameter that is passed with the message.

Control is returned to the caller immediately as this function just queues the message for later processing by the thread.

bool SyntroThread::processMessage(SyntroThreadMsg * msg) [virtual protected]

processMessage is called when there is a message to be processed on the threads message queue. msg contains the message. This function should be overridden in order to process the messages. The function should return true if the messages was processed by the function and no further processing should be performed on this message, false if not.

void SyntroThread::resumeThread() [virtual]

Calling the constructor for SyntroThread doesnt actually start the thread running to allow for the caller to perform any necessary initialization. resumeThread() should be called when everything is ready for the thread to run. resumeThread() calls initThread() and then start(), to get the threads event loop running.
